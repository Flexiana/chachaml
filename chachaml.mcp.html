chachaml.mcp

MCP (Model Context Protocol) server for chachaml.

Exposes experiment runs, models, and artifacts as MCP tools over stdio (JSON-RPC 2.0). Run as a subprocess by an MCP client such as Claude Code, VS Code + Continue, etc.

Start from the command line:

clojure -M:mcp              # uses ./chachaml.db
clojure -M:mcp path/to.db   # custom DB path

Or configure in .claude/mcp.json:

{"chachaml": {
   "command": "clojure",
   "args": ["-M:mcp"],
   "cwd": "/path/to/project"}}

The server reads one JSON-RPC message per line from stdin and writes one response per line to stdout. Logging goes to stderr.

-main

(-main & args)

Entry point: clojure -M:mcp [db-path]. Opens the store and starts the stdio MCP server.

handle-message

(handle-message msg)

Process a parsed JSON-RPC message map and return a response map, or nil for notifications (no :id). This is the testable entry point; the stdio loop in -main is a thin wrapper.

run-server!

(run-server! store in out)

Start the MCP server loop. Reads JSON-RPC from in and writes responses to out. Intended for stdio but also usable with test streams.
